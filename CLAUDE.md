# PyAutoFit Workspace

This is the tutorial and example workspace for **PyAutoFit**, a Python probabilistic programming library for model fitting and Bayesian inference.

## Repository Structure

- `scripts/` - Runnable Python scripts organised by topic
  - `searches/mcmc/` - MCMC search examples (emcee)
  - `searches/mle/` - Maximum likelihood search examples (LBFGS etc.)
  - `searches/nest/` - Nested sampling examples (dynesty, nautilus, ultranest)
  - `features/` - Advanced feature demonstrations
  - `cookbooks/` - API cookbook examples
  - `howtofit/` - Tutorial series scripts
  - `overview/` - Overview examples
  - `simulators/` - Dataset simulation scripts
- `notebooks/` - Jupyter notebook versions of scripts (mirrors `scripts/`)
- `config/` - PyAutoFit configuration YAML files
- `dataset/` - Example 1D datasets
- `output/` - Non-linear search results (generated, not committed)
- `projects/` - Standalone project examples (e.g. cosmology)

## Config

`config/general.yaml` controls search update intervals, output options, logging, profiling, and parallel warnings.

## Shell Commands

- Prefer simple shell commands
- Avoid chaining with `&&` or pipes; run commands separately

## Running Scripts

```
python scripts/overview/overview_1_the_basics.py
python scripts/searches/nest/dynesty/uniform.py
```

## Testing All Scripts

`PYAUTO_TEST_MODE=1` causes non-linear searches to skip actual sampling (fast test run).

Run all scripts using the provided runner:

```
bash run_scripts.sh
```

- Logs for failing scripts are written to `failed/<relative_path>.log`
- Execution continues after each failure
- Final summary shows pass/fail/skipped counts

To run a single script in test mode:

```
PYAUTO_TEST_MODE=1 python3 scripts/overview/overview_1_the_basics.py
```

After a full run, inspect failures:

```
ls failed/
```

### Script ordering rules

1. **`start_here.py` runs first**: if a folder contains `start_here.py`, it is executed before any other scripts in that folder and before descending into subfolders. Some scripts depend on results generated by `start_here.py`.

2. **Skip list from `no_run.yaml`**: `../PyAutoBuild/autobuild/config/no_run.yaml` lists scripts that must not be run, under the `autofit` key. The runner reads this file, prints all skipped scripts with their reasons, and flags any marked `FUTURE_PR` (those with a bug report or GitHub issue that should be fixed in a future PR).

   To add a new skip entry, add a line to `no_run.yaml` under `autofit:`:
   ```yaml
   autofit:
   - ScriptName  # Reason it cannot run yet
   - ScriptName  # bug https://github.com/rhayes777/PyAutoFit/issues/NNN
   ```
   Scripts with a bug link or the word "bug"/"issue"/"fix" in the comment are flagged as `[TODO - should run after a future PR]` in the output.

   Matching is by script basename stem (e.g. `GetDist` matches `plot/GetDist.py`).

## Notebooks

Notebooks in `notebooks/` mirror `scripts/`. Both are kept in sync. Edit the `.py` script first; notebooks are generated.

### Building Notebooks

Notebooks are generated from Python scripts using `generate.py` from the `PyAutoBuild` repo. Run from the workspace root:

```bash
PYTHONPATH=../PyAutoBuild/autobuild python3 ../PyAutoBuild/autobuild/generate.py autofit
```

This converts every `.py` file in `scripts/` to a `.ipynb` in `notebooks/` by:
1. Converting triple-quoted docstrings into `# %%` Jupyter cell markers
2. Running `ipynb-py-convert` to produce `.ipynb` files
3. Restoring commented Jupyter magic commands (e.g. `# %matplotlib` → `%matplotlib`)

Use the `/generate_and_merge` skill to build notebooks, commit, push, raise a PR, and merge into `main`.
